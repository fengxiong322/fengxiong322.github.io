Don't Fear the OOP!

By: Isaac Rosenberg and Feng Xiong

Every setting of the fairy tale has a few key ingredients: princesses, princes, castles and troublemakers. A standard setting will have one castle, will be located in Magic World and will take place sometime around the year 1650.

This description of a fairy tale setting, while not very profound, did two important things: 1) It established the key ingredients for a fairy tale setting and 2) It gave several values that might occur in a default fairy tale setting.

FairyTale setting

  has a certain number of princesses

  has a certain number of princes

  has a certain number of troublemakers

  has a certain number of castles

  is located somewhere

  exists at a certain time

a typical FairyTale setting would have

  castles = 1

  location = Magical World

timePeriod = 1650

It is important to note that FairyTale doesn’t have a space. There is a reason for that. The children reading the story follow a particular format. Spaces confuse them.

Each FairyTale class is different, but have similar properties, called variables, which define how any FairyTale will look. A few examples of variables in our code would be princes, location and timePeriod

public class fairyTaleSetting

{

  int princesses;

  int princes;

  int troublemakers;

  int castles;

  String location;

  int time;

  public fairyTaleSetting ()

  {

     castles= 1;

     location = "Magical World";

     time = 1650;

  }

}

When writing the fairy tale, you are creating a class, not the setting itself (that would be an object). A class is like a (really mixing metaphors here) recipe without any measurements. It says what elements should be in the setting of a fairy tale, but does not say in what amounts.

We have introduced here the idea of subclasses. "Humans"; was a class, and "Magicians" a subclass of it.

The magician can take out any number of rabbits from his hat

This specifies an action, making this different from what we have been doing so far.

takeOutMoreRabbits

  Add one to the number of rabbits

This is what we call a method. This is also what we have been using when returning the variables. The method is described, then you can call this method when you need it

public void takeOutMoreRabbits()

{

  numOfRabbits++;

}

Every time the Magician takes out a rabbit, the number of rabbits increases. In this case, our magician will always pull out only one rabbit each time

If someone asks the magician how many rabbits he has, he will always respond truthfully

Magician Recap

Magicians are based on the idea of humans. They are identical, except that they have some additional qualities, namely a top hat, a beard, a cape, a certain "look". They also look mean and have a certain number of bunny rabbits. The magician can take out any number of rabbits from his hat, increasing the number of rabbits he has. If someone asks the magician how many rabbits he has, he will always respond truthfully. If the magician decides to turn a human into a rabbit, turn the human into a frog and add one to the number of rabbits.

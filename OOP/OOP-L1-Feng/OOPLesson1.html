The Instance Variables

Imagine we have a class, that describe the state of a certain Animal. For instance, its name, if its tired or hungry, and its weight. These things are declared as instance variables. These are put at the top of the class, after the opening line of the class. Instance variables should be private. This is to prevent other classes from making changes that shouldn't be possible. For instance, something may try to multiply the weight of an Animal by 100. This may not make sense. We will talk more about how we can prevent this later.

public class Animal{
  //Instance variables go here
  private String name;
  private boolean tired;
  private double weight;

So far, our class defines fields. They define a section of memory, in which information is to be stored. In general, whenever we use this class, we don't want these fields to be empty. We want to define something in it. To do this, we create a constructor. This constructor will create the Animal. Every Animal is different, so we should give a way to them their own name, and their weight when we create them. We can allow for these changes through parameters. Parameters is information that is given when the constructor is needed

Next, we want our animal to be able to interact with everything around it. For instance, if we feed it food, we want it to not be hungry. Likewise, if it sleeps, we want it to not be tired. We can do this by using methods. Methods define an action to perform.

Finally, we want to be able to run your program. We can do this by declaring a main method. This method is where you use all of the other methods to create a meaningful result. In this example, we make a few different animals

Common Interactions With Classes

Most of the time, Classes should keep their variables private. As we talked about in Exercise 1, this improves the security of your variables. However, we still want to be able to edit these variables, or else there will be no way to interact with the information in the class. So how do we achieve this?

Accessors And Mutators

Everyone, including you, makes use of Accessors and Mutators. The screen that is projecting this text to you is an accessor method. It tells other things what it contains. But the computer would be pretty useless if all it could do was display something. We also want to be able to interact with it. We want to edit something, and the outcome be very predictable. A laptop comes with a keyboard, and a track pad, and you can add other peripherals like a mouse to interact with the laptop. These items are mutators. They mutate the data within the object, and thus causes the accessor method do display something different.

1. Definitions of the following in terms of Objects/classes in Java.

accessor method: A class' accessor method allows another class to read its private variables/fields.

explicit parameter: An explicit parameter is part of the parameter list declared in the constructor.

null reference: A null reference can be given to any non primitive data type, and indicates that it doesn't reference an object.

formal parameter: The parameters inside the parentheses when describing a method.

mutator method: A mutator is a set method, and allows for other classes to edit values.

String literal: The value within double quotes.

pass by value: Passes the actually value and not a reference to the parameter.

actual parameter: The parameters inside the parentheses when calling the method.

implicit parameter: It is the implicit parameter of the object that is calling it.

scope: Represents the time that the variable is "alive".

promoting: When working with PDTs of different sizes, java will automatically grow the smallest data type into the larger one.

pass by reference: Passes the instance variable and not a value to the parameter.

4. Explain the differences between compareTo(), compareToIgnoreCase(), and equals().

compareTo() and compareToIgnoreCase() both try to order two Strings in alphabetical order. However, compareTo() compares ascii values, and so uppercase values are different from the lower case value of the same letter. To solve this problem, compareToIgnoreCase() converts everything to lowercase before comparing. equals() is different because it returns a boolean value. That value indicates if the two strings have the same value, or if they do not.

Allows for polymorphism

CONS

Normally bad programming practise

9. Explain the difference between static/class and instance/member methods.

Static methods do not rely on values created with an object. They are consistent throughout all objects, and can even be used without an object. On the other hand, instance methods are object specific. They require an object in order to be run.

The Instance Variables

Imagine we have a class, that describe the state of a certain Animal. For instance, its name, if its tired or hungry, and its weight. These things are declared as instance variables. These are put at the top of the class, after the opening line of the class. Instance variables should be private. This is to prevent other classes from making changes that shouldn't be possible. For instance, something my try to multiply the weight of an Animal by 100. This may not make sense. We will talk more about how we can prevent this later.

public class Animal{
  //Instance variables go here
  private String name;
  private boolean tired;
  private double weight;

So far, our class defines fields. They define a section of memory, in which information is to be stored. In general, whenever we use this class, we dont want these feilds to be empty. We want to define something in it. To do this, we create a constructor. This constructor will create the Animal. Every Animal is different, so we should give a way to them their own name, and their weight when we create them. We can allow for these changes through parameters. Parameters is information that is given when the constructor is needed
